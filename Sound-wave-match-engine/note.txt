Phase -2 -> FFT -> Fast Fourier Transform: 
Right now the waveform we obtained from phase 1, tells us what is the shape of our wave. At some point in time, it tells us what is the amplitude of the wave. But it doesnt give the following information which specific for some audio track: 
1. Frequency beat (it doesnt tell us about which frequency is begin peaked at that moment. ) -> more specifically which frequencies are active at that moment in time. 
2. Pitch 
3. etc.

So we need these quantities. that is where FFT comes in. This is one of the most important components of DSP (Digital Signal Processing)
If we read theory we find out that fourier transform is nothing but combination of waves (typically sin waves) with varying amplitudes. hence if we apply fast fourier transform we get the components we need. and again. that gives us one while fourier transform which tells if 440Hz exists, 880 Hz exists, etc. but it doesnt tell us at what time did the 440 hz occured. so for that we need multiple fourier transform. 

Here comees SFFT -> short Fourier Transform. Instead of analyzing the whole song as a whole, we pick up tiny windows and analyze them. 
(analogy : Novel -> form 500 pages summary -> misses detail, summary for eveyr 50 pages, misses less details. )

Also inorder to not miss a single detail we perform SFFT on overlapping windows. Before applying FFT directly to the window, we multiply something called Hamming window, it apparently reduces Spectral Leakage. 

The window size we take about is 4096.. which is 4096 / 22.05k = 0.186 sec. each time we look ath this amount of audio. 

the final values which we obtain are stacked over eachotehr to create something called a Spectrogram, it contains the following info: 
1 sample : 50hz yes, 100 hz no.. 
2 sample: ...
etc. 

in the spectrogram, the intensity or loudness is showed with a color. 

Now there is something called time-frequency uncertainty tradeoff. that is if we take bigger windows we will be able to distinguish. 

Again the overlapping frequencies we have already discussed it is to not to miss beat. 

Now there is very imp nuance ehre, when we model the waves windows, we are doing something wrong, that is we are cutting off the signals at places. so that produces a discontinuity and to transform that discontinuity or (warp), FFT introduces more frequency components which are false, this phenomenon is called spectral leakage. to handle this we use something called a hamming window. 

so instead of multiplying the waveform with a rectangular window, we multiply it with a bell shaped curve so as to reduce the discontinuity at the edge, yes that reduces teh edge info but as overlapping is present, thats okay, we dont lose much information. 

FFT assumes the analyzed segment repeats periodically. If the segment has abrupt edges, the repetition introduces discontinuities that create spectral leakage. A Hamming window gradually tapers the signal toward zero at both ends, making the repeated signal smoother and reducing leakage

now we obtain the windows and bins. then map them to the corresponding frequency, mostly this process is completely automised with scipy library, but we have to transform the amplitude to loudness as the higher amplitudes like 1000 may overwhelm smlaler ones like 100 which causes issue in hashing, so keeping the phase 3 in mind, we do this transformation.
